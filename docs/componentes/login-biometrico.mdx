# Login Biométrico

<span className={styles.aliases}>Autenticación biométrica · Identificación biométrica</span>

## Descripción

Componente que permite a usuarios identificarse ante el sistema mediante características físicas o comportamentales únicas (huella dactilar, reconocimiento facial, iris, voz). A diferencia de las credenciales tradicionales que el usuario "sabe" (contraseña) o "tiene" (dispositivo), la autenticación biométrica se basa en lo que el usuario "es". El sistema captura el rasgo biométrico, lo procesa y lo compara contra patrones previamente registrados para validar la identidad.

- Captura características físicas o comportamentales del usuario mediante hardware especializado (sensor de huellas, cámara, micrófono)
- Convierte las características capturadas en representaciones digitales (plantillas biométricas) para su comparación
- Compara la captura contra plantillas previamente registradas para determinar si la identidad coincide

</div>

## Capacidades adicionales

<div className={styles.capBlockOptional}>

<div className={styles.capHeading}>
  <svg viewBox="0 0 16 16" width="16" height="16" fill="currentColor"><path d="M8 1a7 7 0 1 1 0 14A7 7 0 0 1 8 1zm.5 3.5a.5.5 0 0 0-1 0V7.5H4.5a.5.5 0 0 0 0 1H7.5v3a.5.5 0 0 0 1 0V8.5h3a.5.5 0 0 0 0-1H8.5V4.5z"/></svg>
  <span>Capacidades adicionales</span>
</div>

- **Funcionamiento como factor adicional (multifactor):** el componente actúa en conjunto con otra forma de autenticación (contraseña, token), añadiendo una capa biométrica a un flujo previo en lugar de reemplazarlo
- **Autenticación alternativa de respaldo:** permite al usuario autenticarse mediante un mecanismo no biométrico (PIN, contraseña) cuando la verificación biométrica falla o no está disponible
- **Detección de vida (anti-spoofing):** procesa la captura biométrica para verificar que proviene de una persona real y no de una imagen, video o réplica del rasgo

</div>

## Delimitaciones

<Admonition type="danger" title="Qué no es">

- **No** incluye el registro inicial del rasgo biométrico (enrollment), que es parte de la gestión de usuarios
- **No** maneja autorización o permisos posteriores a la autenticación
- **No** es aplicable para autenticación programática entre sistemas
- Requiere infraestructura técnica específica (sensores, algoritmos de matching) no presente en sistemas tradicionales

</Admonition>

## Modalidades

- **Huella dactilar:** escaneo y comparación de patrones únicos de la huella
- **Reconocimiento facial:** análisis de características faciales mediante cámara
- **Reconocimiento de iris:** identificación mediante patrones únicos del iris ocular
- **Reconocimiento de voz:** autenticación mediante patrones vocales característicos

## Dependencias típicas

<div className={styles.dependencies}>

- Hardware adecuado (sensor de huellas, cámara para reconocimiento facial, escáner de iris)
- Motor de comparación biométrica (algoritmos de matching)
- Repositorio de plantillas biométricas registradas ([Base de Datos](/docs/componentes/base-datos))
- Gestor de sesiones o tokens para mantener la autenticación post-verificación
- Mecanismo de autenticación alternativa para casos de falla biométrica ([Formulario de Login](/docs/componentes/formulario-login))

</div>
