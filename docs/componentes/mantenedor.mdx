# Mantenedor

<span className={styles.aliases}>CRUD · ABM · Formulario de gestión</span>

## Descripción

Un mantenedor es una interfaz de usuario que permite gestionar entidades de datos mediante las operaciones básicas sobre información persistente: crear, leer, actualizar y eliminar registros. Su propósito es facilitar que usuarios autorizados interactúen con el repositorio de datos de manera controlada y estructurada.

## Capacidades mandatorias

  <span>Capacidades mandatorias</span>
</div>

- Proporciona acceso directo a las entidades de datos del sistema para su administración completa
- Permite **crear** nuevos registros mediante formularios de entrada
- Permite **visualizar** registros existentes en formato explorable (listas, tablas)
- Permite **editar** registros previamente creados
- Permite **eliminar** registros del sistema
- Valida las entradas antes de modificar el repositorio, manteniendo la coherencia e integridad de los datos
- Restringe las operaciones según el perfil y permisos del usuario autenticado
- Indica qué atributos de la entidad son obligatorios, opcionales o seleccionables de un conjunto predefinido

</div>

## Capacidades adicionales

<div className={styles.capBlockOptional}>

<div className={styles.capHeading}>
  <svg viewBox="0 0 16 16" width="16" height="16" fill="currentColor"><path d="M8 1a7 7 0 1 1 0 14A7 7 0 0 1 8 1zm.5 3.5a.5.5 0 0 0-1 0V7.5H4.5a.5.5 0 0 0 0 1H7.5v3a.5.5 0 0 0 1 0V8.5h3a.5.5 0 0 0 0-1H8.5V4.5z"/></svg>
  <span>Capacidades adicionales</span>
</div>

- **Búsqueda y filtrado:** permite localizar registros específicos mediante texto libre, filtros por campo o combinaciones de criterios
- **Exportación de datos:** genera archivos descargables (CSV, Excel, PDF) a partir de los registros visibles o filtrados
- **Importación masiva:** permite cargar múltiples registros simultáneamente desde un archivo
- **Historial de cambios:** muestra un registro visible de modificaciones realizadas sobre los registros: quién cambió qué, cuándo y desde qué valor anterior

</div>

## Delimitaciones

<Admonition type="danger" title="Qué no es">

- **No** genera cálculos agregados ni estadísticas — eso corresponde a [Reportes](/docs/componentes/reporte) o [Dashboard](/docs/componentes/dashboard)
- **No** es solo un formulario de creación: debe incluir lectura, edición y eliminación para constituir un mantenedor completo
- **No** incluye lógica compleja de negocio ni procesamiento masivo de datos
- **No** define quién tiene permisos para qué operaciones — eso corresponde a [Autorización](/docs/componentes/autorizacion)

</Admonition>

## Modalidades

- **Mantenedor simple.** Gestiona una sola entidad de datos de forma independiente. Es el caso más común: una tabla o listado con su formulario asociado.
- **Mantenedor maestro-detalle.** Gestiona una entidad principal junto con entidades dependientes vinculadas a ella. La interfaz permite navegar de un registro maestro a sus registros de detalle asociados.
