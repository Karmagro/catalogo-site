# Consultas

<span className={styles.aliases}>Búsqueda · Search · Query · Filtrado de información · Information retrieval</span>

## Descripción

Componente que permite a los usuarios recuperar información específica del sistema mediante criterios de búsqueda definidos. Su propósito es facilitar el acceso rápido y dirigido a datos existentes, filtrando el contenido del repositorio según parámetros proporcionados por el usuario.

- Permite al usuario especificar criterios de búsqueda mediante campos, filtros o parámetros
- Recupera registros del repositorio que coinciden con los criterios sin transformarlos ni agregarlos
- Devuelve los resultados en un formato interpretable por el usuario o sistema solicitante

</div>

## Capacidades adicionales

<div className={styles.capBlockOptional}>

<div className={styles.capHeading}>
  <svg viewBox="0 0 16 16" width="16" height="16" fill="currentColor"><path d="M8 1a7 7 0 1 1 0 14A7 7 0 0 1 8 1zm.5 3.5a.5.5 0 0 0-1 0V7.5H4.5a.5.5 0 0 0 0 1H7.5v3a.5.5 0 0 0 1 0V8.5h3a.5.5 0 0 0 0-1H8.5V4.5z"/></svg>
  <span>Capacidades adicionales</span>
</div>

- **Búsqueda avanzada con filtros combinables:** permite restringir la búsqueda mediante múltiples criterios simultáneos (rango de fechas, categoría, autor, etc.) además del texto libre
- **Combinación de datos de múltiples fuentes:** recupera información que requiere relacionar datos de distintas entidades o repositorios del sistema
- **Exportación de resultados:** permite descargar los registros encontrados como archivo (CSV, Excel, JSON), conservando los datos crudos sin agregaciones
- **Paginación:** divide los resultados en páginas para manejar volúmenes grandes sin sobrecargar la interfaz
- **Auto-completado y sugerencias:** propone términos o registros mientras el usuario escribe
- **Historial de búsquedas:** registra y permite reutilizar consultas previas del usuario

</div>

## Delimitaciones

<Admonition type="danger" title="Qué no es">

- **No** procesa ni transforma los datos extraídos: devuelve los registros tal como están almacenados en el repositorio
- Si incluye exportación, exporta los datos crudos filtrados, sin cálculos agregados
- **No** presenta indicadores ni visualizaciones gráficas — eso corresponde a [Dashboard](/docs/componentes/dashboard) o [Reporte](/docs/componentes/reporte)

</Admonition>

## Modalidades

- **Búsqueda por interfaz:** el usuario interactúa con campos y filtros gráficos; típica de búsquedas en sitios web y aplicaciones de usuario final
- **Búsqueda por lenguaje:** el usuario escribe consultas en un lenguaje formal o natural (SQL, GraphQL, lenguaje natural)
- **Búsqueda predefinida:** el sistema ofrece consultas precargadas que el usuario solo dispara, sin componer los criterios

## Dependencias típicas

<div className={styles.dependencies}>

- Repositorio central o [Base de Datos](/docs/componentes/base-datos) sobre la que se realiza la búsqueda
- Sistemas de [Autenticación](/docs/componentes/formulario-login) y [Autorización](/docs/componentes/autorizacion) (cuando los resultados están restringidos por permisos)

</div>
