# Sistema de Notificación

<span className={styles.aliases}>Notificaciones autónomas · Alerta automática · Aviso del sistema · Mensaje automático · Evento notificado</span>

## Descripción

Un sistema de notificación es un componente encargado de entregar mensajes a un usuario, sistema o servicio externo cuando ocurre un evento relevante dentro del sistema de información. Su función es alertar al receptor sobre situaciones que requieren de su conocimiento o acción, permitiendo que el sistema comunique cambios de estado, resultados o condiciones excepcionales sin intervención humana directa. Puede variar en complejidad: desde mecanismos básicos que envían un correo ante un evento puntual, hasta infraestructuras que coordinan múltiples canales y niveles de prioridad.

- Se ejecuta automáticamente ante un evento o condición del sistema, sin intervención del usuario
- Comunica información sintetizada, orientada a la acción o al seguimiento de procesos
- Puede entregar mensajes por múltiples y diversos canales (email, SMS, push, webhook)
- Puede configurarse para distintos públicos (usuarios finales, administradores, sistemas externos)
- Genera mensajes a partir de plantillas y datos variables, personalizando el contenido según el destinatario o evento

## Capacidades adicionales

<div className={styles.capBlockOptional}>

<div className={styles.capHeading}>
  <svg viewBox="0 0 16 16" width="16" height="16" fill="currentColor"><path d="M8 1a7 7 0 1 1 0 14A7 7 0 0 1 8 1zm.5 3.5a.5.5 0 0 0-1 0V7.5H4.5a.5.5 0 0 0 0 1H7.5v3a.5.5 0 0 0 1 0V8.5h3a.5.5 0 0 0 0-1H8.5V4.5z"/></svg>
  <span>Capacidades adicionales</span>
</div>

- **Preferencias del destinatario:** permite a los usuarios configurar qué notificaciones desean recibir, por qué canal y con qué frecuencia (opt-in/opt-out)
- **Agrupación y resumen periódico (digest):** en lugar de enviar cada notificación individualmente, agrupa eventos y los entrega como resumen en intervalos definidos
- **Escalamiento:** reenvía o eleva la notificación si el destinatario no la acusa recibo dentro de un plazo definido
- **Envío manual:** permite a un usuario autorizado disparar una notificación predefinida bajo demanda, seleccionando destinatario y parámetros

</div>

## Delimitaciones

<Admonition type="danger" title="Qué no es">

- **No** sustituye los reportes ni dashboards — eso corresponde a [Reporte](/docs/componentes/reporte) y [Dashboard](/docs/componentes/dashboard)
- **No** debe confundirse con los procesos autónomos que producen la información que origina la notificación
- **No** implica interacción posterior; la respuesta del destinatario es externa al componente

</Admonition>

## Modalidades

- **Transaccional:** confirmación de una acción del usuario (ej.: confirmación de pago, registro exitoso)
- **Promocional:** aviso sobre ofertas o contenido nuevo (ej.: nuevo plan móvil)
- **Alerta:** aviso sobre una condición que requiere atención (ej.: batería baja, umbral excedido)
- **Programada:** enviada en intervalos regulares independiente de un evento puntual (ej.: resumen semanal)
- **Emergencia:** comunicación crítica de alta prioridad que interrumpe al destinatario (ej.: alerta sísmica)
