# Reporte

<span className={styles.aliases}>Report · Informe · Reporte analítico · Reporte de gestión</span>

## Descripción

Componente de salida que extrae, procesa, consolida y presenta información del sistema en un formato estructurado y significativo, orientado a la toma de decisiones. A diferencia de una consulta simple, un reporte aplica transformaciones (cálculos, agregaciones, resúmenes) sobre los datos y los presenta como un documento con propósito específico.

- Consolida y procesa datos mediante cálculos, agregaciones, totales, promedios, conteos u otras operaciones analíticas
- Presenta la información en un formato estructurado con secciones definidas (encabezado, resumen, cuerpo, conclusiones)
- Genera el resultado como un documento producido en un momento específico, no como una vista de actualización continua
- Tiene un propósito específico: responder preguntas de negocio o necesidades de información concretas

## Capacidades adicionales

<div className={styles.capBlockOptional}>

<div className={styles.capHeading}>

- **Parametrización por usuario:** permite ajustar filtros, rangos de fechas, nivel de detalle u otros parámetros antes de generar el reporte
- **Exportación a múltiples formatos:** genera el resultado en formatos descargables (PDF, Excel, Word, HTML)
- **Generación programada y distribución automática:** el sistema genera el reporte según una programación definida y lo distribuye automáticamente a destinatarios predefinidos (por email u otro canal)
- **Inclusión de gráficos y visualizaciones:** incorpora tablas, gráficos de barras, líneas de tendencia u otros elementos visuales junto al texto

</div>

## Delimitaciones

<Admonition type="danger" title="Qué no es">

- **No** es una [Consulta](/docs/componentes/consultas): el Reporte consolida, calcula y agrega información antes de presentarla; la Consulta entrega registros tal como están almacenados
- **No** es un [Dashboard](/docs/componentes/dashboard): el Reporte genera un documento producido en un momento específico, diseñado para ser exportado y analizado; el Dashboard es una vista interactiva de actualización continua
- **No** es una notificación: las notificaciones comunican eventos puntuales; los reportes presentan análisis completos de información
- **No** incluye capacidades de entrada de datos al sistema

</Admonition>

## Modalidades

- **Reporte operacional:** detalla transacciones y actividades recientes (ej.: cartola de movimientos)
- **Reporte analítico:** presenta tendencias, comparaciones y análisis con mayor profundidad
- **Reporte regulatorio:** generado para cumplir con requisitos normativos o legales, con formato y contenido predefinidos
- **Reporte ejecutivo:** resumen de alto nivel orientado a la toma de decisiones gerenciales

## Dependencias típicas

<div className={styles.dependencies}>

- [Base de Datos](/docs/componentes/base-datos) o repositorio central desde donde se extraen los datos
- [Consultas](/docs/componentes/consultas) que recuperan la información necesaria
- [Autenticación](/docs/componentes/formulario-login) y [Autorización](/docs/componentes/autorizacion) para controlar quién puede generar y ver cada reporte
- Servicio de mensajería ([Sistema de Notificación](/docs/componentes/sistema-notificacion)) para distribución automática, cuando esa capacidad está presente

</div>
