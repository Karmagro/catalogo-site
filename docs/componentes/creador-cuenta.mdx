# Creador de Cuenta

<span className={styles.aliases}>Sign-up · Registro de usuario · Formulario de registro</span>

## Descripción

Componente que permite a personas sin cuenta en el sistema crear su primera cuenta de usuario. Presenta una interfaz accesible sin autenticación previa, donde el usuario proporciona sus datos básicos o vincula su identidad mediante otros medios. El sistema valida la información, crea la nueva cuenta y almacena las credenciales de forma segura para que posteriormente puedan ser utilizadas para autenticarse.

- Accesible sin autenticación previa
- Valida los datos de entrada según reglas del sistema (formato de email, fortaleza de contraseña, unicidad de usuario)
- Almacena las credenciales o identificadores del nuevo usuario de forma segura en el repositorio

## Capacidades adicionales

<div className={styles.capBlockOptional}>

<div className={styles.capHeading}>
  <svg viewBox="0 0 16 16" width="16" height="16" fill="currentColor"><path d="M8 1a7 7 0 1 1 0 14A7 7 0 0 1 8 1zm.5 3.5a.5.5 0 0 0-1 0V7.5H4.5a.5.5 0 0 0 0 1H7.5v3a.5.5 0 0 0 1 0V8.5h3a.5.5 0 0 0 0-1H8.5V4.5z"/></svg>
  <span>Capacidades adicionales</span>
</div>

- **Verificación previa por email o SMS:** requiere que el usuario confirme la posesión del medio de contacto declarado antes de activar la cuenta, mediante un código o enlace enviado a su email o teléfono
- **Captcha anti-spam:** incorpora desafíos para evitar registros automatizados por bots
- **Aprobación manual del administrador:** mantiene la cuenta en estado pendiente hasta que un administrador la apruebe explícitamente
- **Restricción por código de invitación:** limita el registro a quienes posean un código previamente emitido, controlando quién puede sumarse al sistema

</div>

## Delimitaciones

<Admonition type="danger" title="Qué no es">

- **No** autentica al usuario después de crear la cuenta — eso corresponde a [Formulario de Login](/docs/componentes/formulario-login)
- **No** modifica cuentas existentes — eso corresponde a [Gestión de Perfil](/docs/componentes/gestion-perfil)
- **No** es administración de cuentas por parte de personal autorizado

</Admonition>

## Modalidades

- **Registro tradicional:** formulario con email o usuario y contraseña
- **Registro con SSO:** "Crear cuenta con Google/Microsoft/Facebook", delegando datos al proveedor
- **Registro en dispositivo:** vinculación de la cuenta al dispositivo mediante huella dactilar, PIN o patrón (común en apps móviles)

## Dependencias típicas

<div className={styles.dependencies}>

- Repositorio de usuarios donde se almacena la nueva cuenta ([Base de Datos](/docs/componentes/base-datos))
- Validador de datos (reglas de negocio sobre contraseñas, emails, etc.)
- Servicio de notificación ([Sistema de Notificación](/docs/componentes/sistema-notificacion) para enviar confirmaciones)
- Captcha o sistema anti-spam (cuando esa capacidad está presente)

</div>
